1. PROJECT OVERVIEW
================================================================================

Project Name: Disaster Risk Awareness System (DisasterAware)
Purpose: An interactive web application to educate users about disaster risks,
         provide safety procedures, and help in emergency decision-making.
Type: Single-Page Application (SPA)
Technology Stack: AngularJS-based Web Application

Target Audience: General public seeking disaster preparedness information
Mobile Support: Yes (Responsive Design)

================================================================================
2. TECHNOLOGY STACK
================================================================================

FRONTEND FRAMEWORK:
  - AngularJS 1.8.2 (Core JavaScript MVC Framework)
  - ngRoute (Routing Module)
  - ngAnimate (Animation Module)

STYLING & UI:
  - CSS3 (Custom styling with CSS Variables)
  - Font Awesome 6.4.0 (Icon Library)
  - Google Fonts: Inter Font Family (wght 300, 400, 500, 600, 700, 800)

STORAGE:
  - Local Storage API (Browser-based data persistence)

MARKUP:
  - HTML5
  - AngularJS Templates (embedded script templates)

NO BACKEND REQUIRED: 
  - Fully client-side application
  - No server-side logic
  - No database required

================================================================================
3. APPLICATION STRUCTURE
================================================================================

PROJECT FILES:
  - index.html   - Main HTML file with templates and Angular logic
  - style.css    - Complete styling (1000+ lines)

ANGULAR COMPONENTS:

Module: disasterApp
  - Dependencies: ngRoute, ngAnimate
  
Controllers:
  1. MainCtrl
     - Global navigation management
     - Mobile menu toggle functionality
     - Route activation tracking
     - Window scroll-to-top on route change

  2. CategoriesCtrl
     - Disaster listing with search/filter
     - Navigate to disaster details

  3. DetailsCtrl
     - Display detailed information for selected disaster
     - Navigation back to categories

  4. HelperCtrl
     - Dynamic form for emergency situation assessment
     - Context-aware advice generation
     - Result display based on user inputs

  5. ChecklistCtrl
     - Track emergency preparedness progress
     - Local storage persistence
     - Progress calculation
     - Reset functionality

Services:
  - DisasterData Service
    * Stores all disaster information
    * Methods: getDisasters(), getDisasterById(id)

4. PAGES & FEATURES
================================================================================

PAGE 1: HOME PAGE (Route: /)
Features:
  - Hero Section with gradient text
  - Call-to-action buttons ("Explore Disasters", "Smart Helper")
  - Feature highlight cards (3 cards):
    * Comprehensive Guides
    * Smart Helper
    * Interactive Checklist
  - Responsive grid layout
  - Animated hero icon with floating effect

PAGE 2: DISASTER CATEGORIES (Route: /categories)
Features:
  - Disaster category grid display
  - Search/Filter functionality (real-time input)
  - Dynamic disaster cards (5 disasters available):
    * Fire / Structure Fire
    * Earthquake
    * Flood
    * Cyclone / Hurricane
    * Medical Emergency
  - Each card displays:
    * Category icon (Font Awesome)
    * Disaster name
    * Risk level badge (High/Moderate/Critical)
    * Description preview (truncated)
    * "Read More" button
  - Color-coded card headers per disaster type
  - Empty state message when no results match search
  - Clickable cards to navigate to details

PAGE 3: DISASTER DETAILS (Route: /disaster/:id)
Features:
  - Back button navigation
  - Disaster header with:
    * Large icon
    * Disaster name
    * Risk level badge (large version)
    * Color-coded background per disaster
  - Four main content sections:
    1. DESCRIPTION
       - Overview of the disaster type
    2. WARNING SIGNS
       - Bulleted list of indicators
       - Trophy icon prefix
    3. PREVENTION TIPS
       - Bulleted list of preventative measures
       - Shield icon prefix
    4. STEP-BY-STEP SURVIVAL GUIDE
       - Numbered steps (up to 7 steps)
       - Animated step indicators
       - Horizontal number badges
  - Error state handling (disaster not found)
  - Responsive two-column layout for warning/prevention sections

PAGE 4: SMART EMERGENCY HELPER (Route: /helper)
Features:
  - Interactive form-based decision helper
  - Three-stage question flow:
    1. Disaster selection (dropdown with 5 options)
    2. Location assessment (radio: Indoors/Outdoors)
    3. Danger level (radio: Yes/No immediate threat)
  - Dynamic form reveal (questions appear as you answer)
  - Contextual advice generation based on:
    * Disaster type
    * Location (indoors/outdoors)
    * Danger level
  - Result card displays recommended actions:
    * Context-aware survival steps
    * Danger-level-based prefix alert
    * Numbered/bulleted format
  - Animated action items reveal
  - Form validation (disabled submit until all fields filled)
  - 5 disaster scenarios × 2 locations × 2 danger levels = 20+ unique combinations

PAGE 5: SAFETY PREPAREDNESS CHECKLIST (Route: /checklist)
Features:
  - Preparedness progress tracker
  - 10-item interactive checklist:
    * Emergency contacts
    * First aid kit
    * Flashlight + batteries
    * Evacuation plan
    * Non-perishable food supply
    * Water supply
    * Medications
    * Emergency radio
    * Important documents
    * Multi-purpose tool
  - Progress bar visualization (percentage-based)
  - Animated progress bar
  - Custom styled checkboxes
  - Local storage persistence (auto-saves on check/uncheck)
  - Reset button with confirmation dialog
  - Completed item styling (strikethrough text)

6. DISASTER DATA MODEL
================================================================================

Each disaster object contains:

{
  id: string,                    // Unique identifier (fire, earthquake, etc.)
  name: string,                  // Display name
  icon: string,                  // Font Awesome icon class
  riskLevel: string,             // "High", "Moderate", or "Critical"
  description: string,           // Detailed description
  warningSigns: array[string],   // List of warning indicators
  preventionTips: array[string], // Preventative measures
  survivalSteps: array[string]   // Step-by-step survival guide
}

AVAILABLE DISASTERS:

1. FIRE / STRUCTURE FIRE
   - Risk Level: High
   - Steps: 7 survival steps
   - Focus: Evacuation, smoke avoidance, emergency protocols

2. EARTHQUAKE
   - Risk Level: High
   - Steps: 6 survival steps
   - Focus: Drop-Cover-Hold, shelter, aftershock awareness

3. FLOOD
   - Risk Level: Moderate
   - Steps: 6 survival steps
   - Focus: Higher ground, water danger, utility safety

4. CYCLONE / HURRICANE
   - Risk Level: High
   - Steps: 6 survival steps
   - Focus: Shelter, eye awareness, evacuation routes

5. MEDICAL EMERGENCY
   - Risk Level: Critical
   - Steps: 6 survival steps
   - Focus: Scene safety, emergency services, CPR/First aid

7. STYLING & DESIGN SYSTEM
================================================================================

CSS ARCHITECTURE:
  - CSS Variables for theming (colors, spacing, shadows)
  - Mobile-first responsive design
  - CSS Grid and Flexbox layouts
  - CSS transitions and animations

COLOR PALETTE:
  - Primary: #2563eb (Blue)
  - Primary Dark: #1d4ed8
  - Secondary: #0f172a (Dark Navy)
  - Background: #f8fafc (Light Gray)
  - Surface: #ffffff (White)
  - Text Main: #1e293b (Dark Gray)
  - Text Muted: #64748b (Medium Gray)
  - Border: #e2e8f0 (Light Border)

RISK COLORS:
  - High: #ef4444 (Red)
  - Moderate: #f59e0b (Amber/Orange)
  - Critical: #dc2626 (Deep Red)

DISASTER THEME COLORS:
  - Fire: #ef4444 (Red gradient)
  - Earthquake: #8b5cf6 (Purple gradient)
  - Flood: #0ea5e9 (Blue gradient)
  - Cyclone: #64748b (Gray gradient)
  - Medical: #ec4899 (Pink gradient)

RESPONSIVE BREAKPOINTS:
  - Desktop: >768px
  - Mobile: ≤768px

TYPOGRAPHY:
  - Font Family: Inter (Google Fonts)
  - Weights: 300, 400, 500, 600, 700, 800
  - Line Height: 1.6 (standard)

SPACING SYSTEM:
  - Padding: 10px, 12px, 16px, 20px, 24px, 28px, 30px, 32px, 40px
  - Gap: 8px, 10px, 12px, 16px, 20px, 24px, 30px, 32px, 40px

BORDER RADIUS:
  - Standard: 12px (var(--radius))
  - Small: 8px
  - Rounded: 30px (search bar)
  - Circle: 50% (badges, step circles)

SHADOWS:
  - Shadow-sm: 1px 3px rgba(0,0,0,0.1)
  - Shadow-md: 4px 6px -1px rgba(0,0,0,0.1)
  - Shadow-lg: 10px 15px -3px rgba(0,0,0,0.1)
  - Box shadows on buttons, cards

ANIMATIONS:
  - fadeIn: 0.6s cubic-bezier
  - slideDown: Vertical slide with fade
  - slideUpFade: Upward slide with fade
  - slideInRight: Horizontal slide from right
  - pulse: 2s infinite scale pulse
  - float: 6s ease-in-out vertical float
  - View transitions: 0.4s scale/opacity
  - Step reveal: Staggered animation delays

UTILITY CLASSES:
  - fade-in: Fade in animation
  - slide-in: Slide down animation
  - float-anim: Floating animation
  - icon-pulse: Pulsing icon animation

  - index.html: ~60KB

13. KNOWN LIMITATIONS
================================================================================

  • Single-language (English only)
  • No backend integration (data is static)
  • No user authentication
  • No multi-user data sync
  • Limits to browser Local Storage capacity (~5-10MB)
  • Checklist data specific to browser (not cloud-synced)
  • No offline-first PWA implementation
  • No accessibility features beyond basic HTML structure
  • No keyboard-only navigation optimizations
  • Voice/screen reader support minimal
